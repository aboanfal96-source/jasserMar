## 🔧 ما تم إصلاحه في هذه النسخة

### 1️⃣ `api/options.js` — محاولات متعددة + رسائل خطأ مفصّلة
- يحاول **أربعة** endpoints مختلفة من Yahoo
- يستخدم headers متطابقة مع متصفح Safari حقيقي
- يرسل تفاصيل الخطأ بدل صمت

### 2️⃣ `api/recommend.js` — محرك تركيبي احتياطي (الأهم!)
عند فشل Yahoo، يحسب العقود محلياً باستخدام:
- **معادلة Black-Scholes** للتسعير
- **التقلب التاريخي** المُستخرج من ATR (يُرسل من المتصفح)
- **معدل الفائدة الخالي من المخاطر** 5%
- **نقاط Strike** كل $1 / $2.5 / $5 / $10 حسب سعر السهم
- **دلتا/جاما** محسوبة من الصيغة الرياضية

⚠️ هذه التقديرات **شفافة**: تظهر شارة صفراء "Yahoo حظر — تقدير تركيبي" في كل عقد تركيبي.

### 3️⃣ `api/test.js` — endpoint تشخيصي جديد
زُره: `your.vercel.app/api/test?symbol=AAPL`

سترى JSON يبيّن:
- ✅/❌ هل يعمل Yahoo Chart؟
- ✅/❌ هل يعمل Yahoo Options؟
- ✅/❌ هل تم ضبط مفتاح Anthropic؟

## 📱 شرح كل تبويب وكيفية استخدامه

### 🏠 رئيسي (Home)
- يعرض **الرسم البياني** للسهم المختار (الأسهم تظهر في الشريط العلوي)
- اضغط أي سهم في الشريط لاختياره
- في النسخة الجوال، تتنقل بين أطر زمنية (يوم/أسبوع/شهر/3 شهور/6 شهور/سنة/سنتين/5 سنوات)

### 📋 قائمة (Watchlist)
- جميع الـ 80+ سهم مُجمّعة بالقطاع
- اضغط أي سهم → يصبح هو السهم المختار في كل التبويبات

### ⚡ تبويب "عقد مرشح" (يظهر مع كل سهم)
**كيف يعمل:**
1. تختار سهم (مثلاً AAPL)
2. النظام يحلل: RSI، MACD، البولينجر، ATR، السمارت موني، 12 استراتيجية أخرى
3. يحسب اتجاه (شراء/بيع/محايد) + درجة فنية + الحركة المتوقعة
4. يرسل هذه البيانات لـ `/api/recommend`
5. الـ API يجلب سلسلة العقود (أو يحسبها تركيبياً)
6. يقيّم كل عقد بـ **7 معايير** بأوزان قابلة للتعلّم
7. يعيد **أعلى عقدين**

**ما تراه:**
- نوع العقد (CALL خضراء ▲ أو PUT حمراء ▼)
- Strike price
- السعر / Delta / IV / OI / حجم
- نقطة التعادل (Break-even)
- Score من 0-100
- تفصيل المعايير السبعة

**الزر المهم: 🧠 تتبع** — اضغطه عندما تريد:
- تسجيل العقد في ذاكرة الذكاء التكيفي
- بعد انتهاء صلاحيته (أسبوع تقريباً) سيُقيَّم تلقائياً
- النتيجة تُعدّل أوزان النظام لمستقبل التوصيات

### 📡 تبويب "رادار" (Liquidity Radar)

**يظهر للسهم المختار فقط** — هذا تصميم مقصود. لماذا؟

> الرادار يحلل **بصمة السيولة الفريدة** لكل سهم: جدران Calls/Puts من بيانات OI الخاصة به، تجمعات السعر التاريخية، Z-Score حجمه. لا معنى لرادار "عام" — كل سهم له شخصية مختلفة.

**ما يعرضه:**
- 🟢 **عقد خضراء** = جدران Call (مناطق جذب صعودي من Open Interest)
- 🔴 **عقد حمراء** = جدران Put (مناطق ضغط هبوطي)
- 🟡 **عقد ذهبية** = منطقة السعر الحالي + تجمعات السيولة السعرية
- ✨ **خطوط متموجة** = محاكاة Smart Money flows
- 🌊 **هالة نابضة** = نشاط حجم غير طبيعي (Z-Score مرتفع)
- 📊 **مؤشر Breakout** أسفل الكانفس = احتمالية الانفجار السعري

**لماذا "محاكاة استنتاجية" وليست بيانات Dark Pool حقيقية؟**
بيانات Dark Pool الفعلية تتطلب اشتراك Unusual Whales أو Cheddar Flow ($800-$1500/شهر). نظامنا يستخدم **بدائل ذكية** من بيانات Yahoo المتاحة (OI حقيقي + Z-Score حجم + S/R clusters). الفرق مذكور بشفافية في تبويب "ℹ️ معلومات".

### 📄 تبويب "عقود" (Recommended Contracts Scanner)
**كيف يعمل:**
1. اضغط زر **"🚀 مسح أفضل العقود الأسبوعية"** (لازم تضغطه — هذا تصميم مقصود لتوفير API calls)
2. النظام يفحص top 24 سهم أمريكي
3. لكل سهم: يستدعي `/api/recommend`
4. يجمع جميع العقود المُرشّحة
5. يرتّبها بـ Score من الأعلى للأقل
6. تظهر القائمة في جدول قابل للتمرير

⏱️ يستغرق المسح ~30-60 ثانية حسب حالة Yahoo.

### 🧠 تبويب "ذاكرة" (Adaptive AI Memory)
**ما يعرضه:**
- **إجمالي/معلق/نسبة الفوز**: 0/0/0% طبيعي عندما تبدأ — لم تتبع أي عقد بعد
- **أوزان المؤشرات**: 7 معايير (فني، ديلتا، IV، OI، حجم، سبريد، حركة) كلها تبدأ بـ 1.00x
- **أوزان الاستراتيجيات**: 12 استراتيجية كلها تبدأ بـ 1.00x

**كيف يتعلم النظام:**
1. اضغط 🧠 تتبع على عقد → يُسجّل في `localStorage` → يصبح "معلق"
2. كل دقيقة، النظام يفحص: هل انتهت الصلاحية؟
3. إذا انتهت: يقارن السعر الحالي بـ Breakeven
   - أعلى من Breakeven (للـ Call) → ربح ✅
   - أدنى → خسارة ❌
4. للمعايير ذات النقاط العالية في العقد:
   - ربح → تزداد أوزانها بـ 6%
   - خسارة → تنقص بـ 6%
5. الأوزان محصورة بين 0.25 و 2.5
6. التوصيات المستقبلية تستخدم الأوزان الجديدة

⚠️ **مهم:** الذاكرة محلية في متصفحك (لا تتزامن بين الأجهزة).

### ⚙️ تبويب "إعدادات"
- إحصائيات أساسية
- زر مسح الذاكرة (يعيد كل الأوزان لـ 1.00x)
- معلومات الإفصاح

## 🚀 خطوات النشر (الأمثل عبر GitHub)

### خطوة 1️⃣: رفع المجلد لـ GitHub
```bash
# فك ملف الـ zip ثم:
cd tadawul-us-v1
git init
git add .
git commit -m "TADAWUL US v1"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/tadawul-us.git
git push -u origin main
```

### خطوة 2️⃣: ربطه بـ Vercel
1. ادخل [vercel.com/new](https://vercel.com/new)
2. **Import** الـ repo
3. **Framework Preset**: اختر **`Other`** (ليس Next.js!)
4. **Root Directory**: اتركه فارغ (`./`)
5. **Build Command**: اتركه فارغ
6. **Output Directory**: اتركه فارغ
7. اضغط **Deploy**

### خطوة 3️⃣: إضافة مفتاح Claude API
1. بعد النشر: **Settings** → **Environment Variables**
2. أضف:
   - **Name**: `ANTHROPIC_API_KEY`
   - **Value**: مفتاحك من [console.anthropic.com](https://console.anthropic.com)
   - **Environments**: اختر الثلاثة (Production, Preview, Development)
3. ارجع لـ **Deployments** → اضغط ⋯ على آخر deployment → **Redeploy**

### خطوة 4️⃣: التحقق من النجاح
افتح هذه الروابط بالترتيب:

```
https://YOUR.vercel.app/api/test?symbol=AAPL
```
سترى تشخيص كامل (ماذا يعمل وماذا لا).

```
https://YOUR.vercel.app/api/stock?symbol=AAPL
```
بيانات السهم.

```
https://YOUR.vercel.app/api/options?symbol=AAPL
```
سلسلة العقود (إذا فشل، تتفعل المحاكاة التركيبية تلقائياً عند طلب توصية).

```
https://YOUR.vercel.app/
```
الواجهة الكاملة (سطح المكتب).

```
https://YOUR.vercel.app/mobile
```
واجهة الجوال.

## 🐛 المشاكل الشائعة

### "Yahoo حظر سلسلة العقود — تقدير تركيبي"
**هذا طبيعي تماماً.** Vercel على عناوين IP مشتركة وYahoo قد يحجبها أحياناً. النظام تلقائياً يستخدم Black-Scholes ويُعلن ذلك بشفافية.

### العقود لا تظهر إطلاقاً
1. تحقق من `/api/test` — هل Yahoo Chart يعمل؟
2. إذا كل شيء فاشل: انتظر 5 دقائق وأعد المحاولة (حد سرعة Yahoo)
3. جرّب من **region مختلف** في Vercel (Settings → Functions → Region)

### الذكاء الاصطناعي (Claude) لا يجيب
- تأكد من إضافة `ANTHROPIC_API_KEY` و**Redeploy**
- زُر `/api/test` للتأكد من configured: true

### الواجهة بطيئة على الجوال
- النسخة الجوال خفيفة بطبيعتها
- إذا تأخرت: السبب طلبات Yahoo (تحدّث كل 5 ثوان)
